## OKR 会议制度

### 每次对话开始时（1分钟）
```
1. 翻看 OKR.md 检查是否有关联 KRs 待完成
2. 查看 ERROR_LOG 避免重复错误
3. 确认数据路径存在、脚本可运行
```

### 每次对话结束时（2分钟）
```
1. 更新关联的 KR 状态（✅/⚠️/⬜）
2. 如有报错 → 记入 ERROR_LOG
3. 写 artifact（策略/复盘/验证均写）
4. 确认下次需要关注的 KRs
```
