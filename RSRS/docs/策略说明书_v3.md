# RSRS + C63 复合动量策略
**版本 v3 | 2026-07-11 | 定稿稳定版**

## 一、策略概述

本策略是一个**三层架构**的A股ETF择时轮动系统，通过三层独立信号协同工作，实现"大盘牛熊判断 → 强势品种选择 → 仓位动态管理"的全流程自动化交易。

```
┌─────────────────────────────────────────────────────┐
│                   三层架构                             │
│                                                      │
│  Layer 1: RSRS大盘择时    → 进出场时机                │
│           (沪深300 z-score ≥0.7 买入, ≤-1.0 卖出)     │
│                                                      │
│  Layer 2: C63动量选品      → 买什么                  │
│           (63日收益率排名，选最强那只)                  │
│                                                      │
│  Layer 3: 波动率仓位管理   → 买多少                  │
│           (目标年化波动率 16%)                        │
└─────────────────────────────────────────────────────┘
```

**核心哲学**：
- 择时层判断"市场是否值得参与"
- 动量层找出"当前最强品种"
- 仓位层控制"不要下太重的注"

## 二、三层机制详解

### Layer 1 — RSRS大盘择时

**RSRS（Resistance and Support Relative Strength）** 的核心思想：

> 高点对低点的回归斜率（β值），反映了当前市场"支撑力"与"阻力"的相对强弱。
> 当市场上涨顺畅（低点抬升>高点抬升），β趋近1 → 多头信号
> 当市场下跌顺畅（高点降低>低点降低），β趋近0 → 空头信号

**计算步骤：**

```
第1步：每日用过去18天的高点序列对低点序列做OLS回归
        high = β × low + α
        → 得到每日β值（斜率）

第2步：用过去900天的β值序列计算z-score
        z = (当前β - 900天均值) / 900天标准差
        → 标准化后消除β值漂移的影响

第3步：根据z-score阈值触发信号
        z ≥ +0.7  → 买入（做多模式开启）
        z ≤ -1.0  → 卖出（空仓保护）
        中间区域  → 保持现有状态（不追不砍）
```

**为什么用0.7和-1.0不对称？**

- 买入阈值0.7较高 → 只在市场明显强势时进场，减少"假突破"
- 卖出阈值-1.0较低 → 给空头一点余地，避免震荡市被反复收割
- 这是经过参数扫描验证的最优不对称组合

**RSRS的本质**：不预测涨跌，只判断"当前市场是否处于有利于多头的状态"。

### Layer 2 — C63动量选品

**动量效应的核心假设**：近期涨幅最大的资产，未来短期表现仍倾向于优于其他资产。

**本策略采用单窗口63日动量，理由：**

| 窗口 | 优势 | 劣势 |
|------|------|------|
| 21日 | 灵敏 | 噪音大、频繁跳车 |
| 42日 | 适中 | 略滞后 |
| **63日** | **兼顾灵敏度与稳定性** | 基本无明显劣势 |
| 126日 | 稳定 | 过于滞后，错过主升 |

**选品规则：**

```
1. 在10只ETF池中，计算每只ETF的63日收益率
2. 排除负收益标的（只选正动量）
3. 收益率最高的那只 → 即为当期持仓标的
4. 每次调仓日重新排名
```

**只选1只（Top1）的理由**：动量效应有强者恒强特性，集中持有最强标的可以最大化动量收益。分散持有（Top2+）会稀释收益，经回测验证。

### Layer 3 — 波动率仓位管理

**核心思想**：波动越大的资产，下注比例越小，让组合整体波动率维持在目标水平。

**计算方法：**

```
目标波动率 = 16% 年化
仓位 = 16% / (ETF 70日年化波动率)
上限 = 90%（不做空）
下限 = 10%（市场太危险时空仓到10%）
最低变动阈值 = 5%（仓位变化<5%时不调仓）
```

**示例**：

| ETF | 70d波动率 | 计算仓位 | 实际仓位 |
|-----|----------|---------|---------|
| 沪深300（熊市） | 8% | 16%/8%=200% | 90%（上限） |
| 科创50（高波动） | 32% | 16%/32%=50% | 50% |
| 黄金（低波动） | 10% | 16%/10%=160% | 90%（上限） |

**目标16%的依据**：
- 桥水等机构的风险平价策略，目标波动率区间通常为10%~20%
- 回测中16%是夏普最优值（最优Calmar比）
- 可以有效将最大回撤控制在15%左右

### 锁仓机制（特别说明）

**规则**：买入后42天内，即使RSRS翻空，也不卖出。

**逻辑**：
1. RSRS设计初衷是"市场状态"判断，不是"方向预测"
2. 市场短期波动（几天~几周）往往由噪音驱动，RSRS信号会反复抖动
3. 锁仓42天=约2个月，可以让噪音平息，过滤掉短期假信号

**RSRS信号与锁仓的关系（两种机制独立运行）：**

```
┌──────────────────────────────────────────────────────┐
│ 持仓中（锁仓未到期）                                    │
│  → RSRS翻空 → 不操作（锁仓保护）                        │
│  → 锁仓到期 + RSRS仍多头 → 继续持有                     │
│  → 锁仓到期 + RSRS已翻空 → 清仓                        │
│                                                      │
│ 空仓中                                                │
│  → RSRS ≥ 0.7 → 买入                                │
│  → RSRS < 0.7 → 等待                                 │
└──────────────────────────────────────────────────────┘
```

**锁仓42天 vs 调仓周期RB=42天**：
- 锁仓：防止被假信号甩下车
- 调仓：重新计算C63排名，看是否有更好的标的

## 三、完整交易规则

### 日常运行流程

```
每个交易日：
  1. 获取沪深300RSRS z-score
     → z ≥ 0.7 → 进入/保持做多模式
     → z ≤ -1.0 → 进入/保持空仓模式（需锁仓到期才实际清仓）
     → 中间 → 保持现有状态

  2. 查询C63动量排名
     → 若调仓日到达（距上次调仓≥42天）→ 重新选Top1
     → 若有更好标的 → 换仓
     → 若锁仓未到期 → 不换仓

  3. 查询波动率，计算目标仓位
     → 计算仓位 → 若与当前仓位差异≥5% → 调仓
```

### 调仓日判断

```
上次调仓日 = T
当前日 = today
if today - T ≥ 42天：
    → 今天调仓，重新选C63动量Top1
else:
    → 今天不调仓，保持现有持仓
```

## 七、稳健性警示

### ⚠️ 已知局限性

**1. 置换检验存疑**
- 将RSRS信号随机打乱50次后，平均收益（+28%）不低于真实策略
- 暗示：RSRS择时层可能仅过滤了大熊市，对日常牛熊转换的预测价值有限
- C63动量层可能是更主要的收益来源

**2. 数据范围有限**
- 仅覆盖2018~2026，约2个牛熊周期
- 样本量偏小，统计显著性有限

**3. 2022熊市表现平淡**
- 熊市期间仅+0.0%，虽然避开了大跌，但没有带来正收益
- 空仓保护有效果但不如预期理想
