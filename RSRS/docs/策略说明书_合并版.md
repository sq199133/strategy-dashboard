# RSRS + C63 复合动量策略
**版本 v3 | 定稿日期 2026-07-11 | 合并日期 2026-07-19**

---

## 一、策略概述

本策略是一个**三层架构**的A股ETF择时轮动系统，通过三层独立信号协同工作，实现"大盘牛熊判断 → 强势品种选择 → 仓位动态管理"的全流程自动化交易。

```
┌─────────────────────────────────────────────────────┐
│                   三层架构                             │
│                                                      │
│  Layer 1: RSRS大盘择时    → 进出场时机                │
│           (沪深300 z-score ≥ 0.7 买入, ≤ -1.0 卖出)   │
│                                                      │
│  Layer 2: C63动量选品      → 买什么                  │
│           (63日收益率排名，选最强那只)                  │
│                                                      │
│  Layer 3: 波动率仓位管理   → 买多少                  │
│           (目标年化波动率 16%)                        │
└─────────────────────────────────────────────────────┘
```

**核心哲学**：
- 择时层判断"市场是否值得参与"
- 动量层找出"当前最强品种"
- 仓位层控制"不要下太重的注"

---

## 二、三层机制详解

### Layer 1 — RSRS大盘择时

**RSRS（Resistance and Support Relative Strength）** 的核心思想：

> 高点对低点的回归斜率（β值），反映了当前市场"支撑力"与"阻力"的相对强弱。
> 当市场上行顺畅（低点抬升 > 高点抬升），β趋近1 → 多头信号
> 当市场下行顺畅（高点降低 > 低点降低），β趋近0 → 空头信号

**计算步骤：**

```
第1步：每日用过去18天的高点序列对低点序列做OLS回归
        high = β × low + α
        → 得到每日β值（斜率）

第2步：用过去1200天的β值序列计算z-score
        z = (当前β - 1200天均值) / 1200天标准差

第3步：根据z-score阈值触发信号
        z ≥ +0.7  → 买入（做多模式开启）
        z ≤ -1.0  → 卖出（空仓保护）
        中间区域  → 保持现有状态（不追不砍）
```

**为什么用0.7和-1.0不对称？**

- 买入阈值0.7较高 → 只在市场明显强势时进场，减少"假突破"
- 卖出阈值-1.0较低 → 给空头一点余地，避免震荡市被反复收割

**RSRS的本质**：不预测涨跌，只判断"当前市场是否处于有利于多头的状态"。

---

### Layer 2 — C63动量选品

**动量效应的核心假设**：近期涨幅最大的资产，未来短期表现仍倾向于优于其他资产。

**选品规则：**

```
1. 在11只ETF池中，计算每只ETF的63日收益率
2. 排除负收益标的（只选正动量）
3. 收益率最高的那只 → 即为当期持仓标的
4. 每次调仓日（RB=42天）重新排名
```

**只选1只（Top1）的理由**：动量效应有强者恒强特性，集中持有最强标的可以最大化动量收益。分散持有（Top2+）会稀释收益，经回测验证。

---

### Layer 3 — 波动率仓位管理

**核心思想**：波动越大的资产，下注比例越小，让组合整体波动率维持在目标水平。

```
目标波动率 = 16% 年化
仓位 = 16% / (ETF 70日年化波动率)
上限 = 90%（永远不满仓）
下限 = 10%（最危险时空仓到10%）
最低变动阈值 = 5%（仓位变化<5%时不调仓）
```

**目标16%的依据**：桥水等机构风险平价目标区间10%~20%，回测中16%是夏普最优值。

---

### 锁仓机制（特别说明）

**规则**：买入后42天内，即使RSRS翻空，也不卖出。

**逻辑**：RSRS设计初衷是"市场状态"判断，不是"方向预测"。短期波动往往由噪音驱动，锁仓42天可以让噪音平息。

```
持仓中（锁仓未到期）：
  → RSRS翻空 → 不操作（锁仓保护）
  → 锁仓到期 + RSRS仍多头 → 继续持有
  → 锁仓到期 + RSRS已翻空 → 清仓

空仓中：
  → RSRS ≥ 0.7 → 买入
  → RSRS < 0.7 → 等待
```

## 三、完整交易规则

```
每个交易日：
  1. 获取沪深300RSRS z-score
     → z ≥ 0.7 → 进入/保持做多模式
     → z ≤ -1.0 → 进入/保持空仓模式（需锁仓到期才实际清仓）
     → 中间 → 保持现有状态

  2. 查询C63动量排名
     → 若距上次调仓 ≥ 42天 → 重新选Top1
     → 若有更好标的 → 换仓
     → 若锁仓未到期 → 不换仓

  3. 查询波动率，计算目标仓位
     → 若与当前仓位差异 ≥ 5% → 调仓
```

## 四、核心参数（永久锁定）

| RSRS N（回归窗口） | **18** | 日内高低点回归窗口 |

| 仓位上限 | **90%** | 永远不满仓 |

## 八、排除的优化方向（已验证无效）

### RSRS修正方向 — 全部放弃
- ❌ 方向乘数（β×close方向）→ beta均值归零，信号失效
- ❌ 9+9分段 → 高低点比较无额外信息
- ❌ MA平滑（MA=3~60）→ 破坏high/low统计基础
- ❌ 趋势线斜率/斜率积 → 2022年虚假高收益
- ❌ 价格通道/close斜率/残差差 → 全面劣于原版

### 全球RSRS替代方案 — 全部放弃
- ❌ 全球合成RSRS（4市场平均）：CAGR +12.1%，Sharpe 0.69
- ❌ 标普500 RSRS单独：CAGR -4.7%
- ❌ 全池平均RSRS：CAGR +7.0%，信号过于迟钝

### 标的池扩展 — 全部放弃
- ❌ 价值/质量/基本面/现金流因子ETF：回测无增益，过拟合风险高
- ❌ 因子ETF（如159399现金流）：仅333天数据，CAGR虚高6.6%是过拟合

## 九、策略维护

### 每日/每周
- 查看RSRS z-score是否触及 0.7（买入）或 -1.0（卖出）
- 检查锁仓是否到期（买入后42天）

### 月度
- 确认数据文件更新（各ETF日线数据）
- 运行 `current_advice.py` 获取信号摘要

### 年度
- 检查ETF池标的是否仍可交易、流动性是否足够
- 评估当年绩效是否符合预期范围

- 历史日线：`D:\QClaw_Trading\data\history/`
