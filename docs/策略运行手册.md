## 快速开始

### 每周例行操作

**每周一执行以下步骤：**

#### 步骤 1：运行周线扫描

```bash
cd D:\QClaw_Trading
python3 weekly_scan_v4.py
```

输出示例：
```
[扫描日期] 2026-07-07
候选 ETF 数量: 12
========================================
排名  代码      名称        综合得分  价格   MA21    偏离%
  1  512480   半导体ETF   0.0521   1.234  1.189  +3.8%
  2  159995   芯片ETF     0.0488   1.567  1.521  +3.0%
...
========================================
推荐: 512480 半导体ETF
信号强度: 强（得分 0.0521，前5均值 0.0412）
```

#### 步骤 2：确认信号并执行交易

- **强信号**（推荐标的得分明显高于次名）：直接执行
- **弱信号**（得分接近）：参考偏离度和赛道分布判断
- **无信号**（候选为空）：空仓等待，不追高

#### 步骤 3：持仓检查（如已有持仓）

运行前检查：
- 当前持仓标的最近一周是否有 MA21 跌破风险？
- 持仓标的 ATR 是否持续下降（波动力度衰减）？

## 数据维护

### ETF 池更新

ETF 池文件：`D:\QClaw_Trading\data\etf_pool_V1_full.json`

新增 ETF 时需要：
1. 添加 ETF 代码和分类到 JSON
2. 下载对应历史数据到 `data\history_long_v2\`
3. 运行周线扫描验证数据完整性

### 历史数据格式

每个 ETF 文件为 `code.json`，格式：
```json
{
  "code": "512480",
  "name": "半导体ETF",
  "records": [
    {"w": "2026-W27", "date": "2026-07-04", "close": 1.234,
     "open": 1.220, "high": 1.245, "low": 1.215, "vol": 1234567},
    ...
  ]
}
```

**注意**：
- `close` 为周五收盘价（用于信号计算）
- `vol` 为当周成交量
- 历史数据从 2016 年开始

## 策略参数调整指南

### 什么时候需要调整参数？

| 情况 | 建议 |
|:---|:---|
| 连续 3 个月跑输大盘 | 检查 max_dev 是否过于严格 |
| 交易频率过高（>3次/月） | 检查 ATR_FACTOR 是否过于宽松 |
| 亏损集中在单次大回撤 | 收紧止损（-8% → -6%） |
| 长期无信号（>3周） | 放宽 ATR_FACTOR（0.85 → 0.75） |

### 禁止随意调整的情况

- 回测 IS 阶段表现良好，OOS 阶段短期波动 → **不调**，这是正常波动
- 仅凭 1-2 个月表现判断 → **不调**，数据量不足
- 调整后 IS 变好但 OOS 变差 → **不调**，说明过拟合

## 常见问题

**Q: 扫描结果为空怎么办？**  
A: 说明当前无 ETF 同时满足 MA21 硬过滤 + ATR 条件。这是正常现象，说明市场整体处于弱势或分化状态，空仓等待即可。

**Q: 推荐标的和持仓标的冲突怎么办？**  
A: 以扫描信号为准。若推荐标的与持仓不同，说明原持仓打分已下降，应换仓。

**Q: 持仓中出现止损信号但我看好后市怎么办？**  
A: 严格按策略规则执行止损。策略的核心是系统性，不允许主观判断_override。止损后可在下一周重新评估是否入场。

**Q: 历史数据缺失某 ETF 怎么办？**  
A: 该 ETF 不会被纳入扫描池。补充数据后再使用。

**Q: 周末（周六/周日）可以提前运行扫描吗？**  
A: 可以，扫描使用已收盘的历史数据，周末运行结果与周五收盘后一致。

## 风险提示

- 历史回测不代表未来表现
- 策略在市场风格切换时可能表现不佳（如 2018 年熊市）
- 单只持仓波动较大，建议仓位管理
- 最大回撤历史上达到 -27.4%（OOS），需有心理准备
