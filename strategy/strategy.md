### 3.1 相关性计算规范（强制执行）

**计算方法：**
- **指标**：Pearson 相关系数，基于日收益率（不含分红）
- **数据窗口**：最近 **120 个交易日**（约6个月，覆盖完整涨跌周期）
- **公式**：r = Cov(R_a, R_b) / σ_a × σ_b
- **实现脚本**：`D:\QClaw_Trading\corr_etf.js`（必用，不得人工估算）

**为什么用日收益率而非收盘价？**
- 收盘价相关性会虚高（两只ETF都跟着大盘涨，相关系数会掩盖真实关系）
- 日收益率直接反映两只ETF在同一交易日的同步涨跌程度，更准确

**具体计算流程（每轮调仓必须执行）：**

```
输入：候选ETF代码列表（含新候选 + 现有持仓）
      当前持仓代码列表
输出：候选ETF maxCorr 报告（每只候选与所有持仓的最大相关系数）

步骤：
1. 对每只ETF，用腾讯接口拉取最近120日日K（shXXXXXX格式，前复权）
2. 计算每日收益率序列：R_t = (P_t - P_{t-1}) / P_{t-1}
3. 对每对 ETF_a × ETF_b，计算 Pearson(R_a, R_b)
4. 对每个候选，maxCorr = max(r vs 持仓1, r vs 持仓2, ...)
5. 若数据不足60个共同交易日 → 标记"数据不足" → 默认排除
```

**❌ 严禁行为（v3.4 新增）：**
- 严禁凭直觉或"看起来走势差不多"估算相关性
- 严禁用"不同指数名称不同行业"推断相关性低
- 严禁用"≈0"、"几乎不相关"等模糊表述替代真实计算
- 违者后果：持仓集中暴露，风格偏移，策略失效

**✅ 正确行为：**
- 每次调仓前，运行 `corr_etf.js`，打印完整矩阵
- 将矩阵截图/数据纳入复盘报告
- 人工决策时必须引用具体 r 值（如"r=0.735，需排除"）

### 3.2 相关性判断标准

| 相关系数范围 | 含义 | 操作 |
|:------------:|------|------|
| r > 0.85 | 🔴 高度重叠，实际等同 | 必排除（只保留评分最高/规模最大的）|
| 0.70 < r ≤ 0.85 | 🔴 高相关 | 排除 |
| 0.50 < r ≤ 0.70 | 🟡 中高相关 | 结合其他因素判断（星级差≥2可保留）|
| 0.30 < r ≤ 0.50 | 🟢 中低相关 | 理想分散配置，优先纳入 |
| r ≤ 0.30 | ⚪ 低相关 | 最优分散效果，强烈推荐 |

**阈值（v3.4 确认）：maxCorr ≤ 0.70 才可纳入持仓。**

### 3.3 选池优先级（v3.7 修订，重要）

> **核心原则（v3.7 新增）：** 上涨预期优先，相关性其次。

**操作步骤：**

```
1. 【按上涨预期排序】先按星级（降）→ 20日涨幅（降），对全市场所有ETF排序
2. 【保留最强信号】选出星级最高的前N个（≥⭐⭐⭐⭐）
3. 【用相关性筛选】在强信号之间检查maxCorr，剔除>0.70的
4. 【填满5只】循环直到持仓满5只
```

> ❌ **旧规则（v3.6 及之前，已删除）：** 先检查相关性，优先选相关性低的标的 → 导致选入了信号弱的ETF（如黄金/纳指），反而跑输大盘。
>
> ✅ **新规则（v3.7）：** 先保证信号强度，再在强信号中做相关性筛选。信号强但相关性高 → 换另一只强信号；信号弱但相关性低 → 不买。

**等信号优先级原则：**

| 场景 | 操作 |
|------|------|
| 多只⭐⭐⭐⭐⭐信号 | 用相关性选，分散到不同市场 |
| ⭐⭐⭐⭐信号 vs ⭐⭐⭐信号 | 优先⭐⭐⭐⭐（更强趋势） |
| 两信号星级相同 | 优先20日涨幅更高的 |
| 同市场两只高星信号 | 只选评分最高/规模最大的那只 |

### 统一规则（消除2.4节与2.5节矛盾）

- ✅ **maxCorr > 0.70** → 直接排除（不在星级基础上降1星）
- ✅ **星级优先**：星级高的标的优先被考虑

### 3.4 年化50%的持仓质量要求（v3.4 新增）

> 目标：年化 ≥50%，每1.5年翻一倍。

**必要条件（倒推计算）：**

| 指标 | 计算公式 | 要求 | 说明 |
|------|---------|:----:|------|
| 平均持仓收益率 | (1+r)^n - 1 ≥ 10% | r ≥ 10%/次 | 每次建仓期望收益≥10% |
| 持仓胜率 | 盈利次数 / 总交易次数 | ≥ 55% | 盈利覆盖亏损有余 |
| 年均盈利交易次数 | ≥ 6次/年 | - | 每季度至少1-2次有效建仓 |
| 单笔最大亏损 | ≤ -5% | 硬止损 | 趋势破坏立即出，不扛单 |
| 最大组合回撤 | ≤ -15% | 硬底线 | 回撤>15%全面检视策略 |
| 持仓ETF平均星级 | ≥ 4星 | - | 4星以上标的才有足够爆发力 |
| 趋势行情捕获率 | 趋势段涨幅 / 指数涨幅 | ≥ 80% | 抓到趋势时必须重仓持有 |

**复盘验证（每次必须检验）：**

```
年化收益率 = (1 + 平均持仓收益率)^年均盈利交易次数 × (1 - 平均亏损率)^年均亏损交易次数 - 1

例：平均持仓收益率15%，胜率55%，年均10笔交易
= (1.15^5.5) × (0.95^4.5) - 1 ≈ 1.24 × 0.80 - 1 ≈ -0.8%  ← 失败
= (1.15^7) × (0.97^3) - 1 ≈ 2.66 × 0.91 - 1 ≈ 142%  ← 成功

结论：平均收益率15% + 胜率55% + 平均亏损<3% 才能实现年化50%+
```

> ⚠️ **实盘复盘时**，必须把每只持仓的实际收益率和胜率代入公式，检验是否在年化50%轨道上。

## 四（附）、追高风险过滤机制（v3.8 新增，必检）

> **设计原理：** MA20趋势确认 + MACD金叉均为**滞后信号**。当四个买入条件全部满足时，价格往往已偏离MA20一段距离。此时买入，一旦遇到正常的价格回踩MA20，就会被套亏损。追高过滤的核心目标是：**在信号有效的前提下，回避买入时机不佳的风险**。

> ⚠️ **执行顺序：** 先按第二节计算原始评分 → 再按本节执行追高过滤（扣分或禁买） → 得到最终星级。

---

### 4A.1 三维度追高检测

| 维度 | 计算方式 | 数据来源 | 意义 |
|------|---------|---------|------|
| **乖离率 BIAS20** | (收盘价 - MA20) / MA20 × 100% | 日K线 | 价格偏离均线的程度，偏离越大回归压力越大 |
| **短期涨幅** | 3日累计涨幅、5日累计涨幅 | 日K线 | 短期加速上涨，过快上涨后必回调消化 |
| **连涨天数** | 连续收盘价 > 前一日收盘价的天数 | 日K线 | 持续上涨的惯性耗尽概率 |

### 4A.2 🟡 黄线条件（触发→扣分）

| # | 条件 | 扣分 | 说明 |
|:--:|------|:----:|------|
| Y1 | BIAS20 > 5% 且 ≤ 8% | -1分 | 价格偏高但未极端 |
| Y2 | 3日累计涨幅 5%~10% | -1分 | 短期偏热 |
| Y3 | 5日累计涨幅 10%~15% | -1分 | 短期过热 |
| Y4 | 连续上涨 5~7天 | -1分 | 上涨惯性消耗中 |

**黄线叠加规则：**
- 触发1个黄线 → 原始评分 -1分
- 触发2个黄线 → 原始评分 -2分
- 触发3个及以上黄线 → 等同🔴红线，禁止买入

### 4A.3 🔴 红线条件（触发→禁止买入，无论几星）

| # | 条件 | 阈值 | 说明 |
|:--:|------|------|------|
| R1 | **BIAS20 > 8%** | 绝对偏离 | 价格离均线太远，回归概率极高 |
| R2 | **3日涨幅 > 10%** | 短期暴涨 | 3天涨10%以上的ETF，90%会在5日内回调 |
| R3 | **5日涨幅 > 15%** | 短期极端 | 几乎所有资产涨15%后都需要消化 |
| R4 | **连续上涨 ≥ 8天** | 连涨极限 | 连涨8天以上，技术性回调几乎是必然 |
| R5 | **黄线叠加 ≥ 3个** | 综合过热 | 多维度同时过热，风险极高 |

> ⚠️ **红线是硬性禁止，不可绕过。** 即使⭐⭐⭐⭐⭐满分13分，触发红线也不买入。

### 4A.4 观察池机制

**触发场景：** 标的满足全部买入条件（四星以上）但触发🔴红线。

**操作流程：**

```
1. 标的触发红线 → 加入【观察池】，不执行买入
2. 每日复盘时检查观察池标的：
   - BIAS20 是否回落到 ≤ 5%
   - 3日/5日涨幅是否回落到正常范围
   - 连涨是否中断
3. 回落条件满足（BIAS20 ≤ 5% AND 无其他红线）→ 恢复买入资格
4. 回落到 BIAS20 ≤ 2%（价格贴近MA20）→ 最佳买点（"回踩买入"）
5. 观察池最长保留 15个交易日，超时自动移除
```

**观察池在复盘报告中的格式：**

```
【观察池】
| 代码 | 名称 | 入池日期 | 触发红线 | 当前BIAS20 | 当前星级 | 状态 |
|------|------|---------|---------|:--------:|:-------:|------|
| 159915 | 创业板ETF | 04-22 | R1(9.2%) | 6.1% | ⭐⭐⭐⭐ | 观望 |
| 510300 | 沪深300ETF | 04-23 | R2(11.5%) | 3.8% | ⭐⭐⭐⭐⭐ | ✅可买 |
```

## 五、卖出规则（v3.2核心修订）

> **回测结论（沪深300ETF，2018-2026，约8年）：**
> - 仅用MA20跌破：**年化1.05%，最大回撤16.20%，16笔交易**
> - 加MACD死叉：年化0.60%，回撤14.48%，18笔 → 收益更低，次数更多
> - 加连续N天绿柱：年化0.53%-1.05%，均不及仅用MA20
> - **结论：MACD所有卖出变体均劣于仅用MA20跌破，予以删除**

**触发条件（满足任一立即离场）：**

| # | 条件 | 处理 |
|:--:|------|------|
| ① | **收盘价 < MA20**（当天收盘跌破20日均线） | 第二天开盘卖出 |
| ② | **硬性止损**（亏损 > 5%） | 当天无条件卖出 |

**删除的旧规则：**
- ❌ ~~MACD死叉~~ → 经验证，比仅MA20跌破更差
- ❌ ~~连续3天绿柱~~ → 回测证明劣于仅MA20跌破
- ❌ ~~MACD红柱转绿~~ → 与MACD死叉100%重复

**v3.9 新增：止损后冷静期（冷却规则）**
> 触发原因：港股创新药ETF止损后（亏损-6.32%），4/24当日立即买入深证50，介入时机风险较高。连败场景下（止损→立即追入另一个弱势标的）会快速消耗账户资金。

| 场景 | 处理 |
|------|------|
| 止损执行后 | 若市场无明确⭐⭐⭐⭐以上新信号，等待**2-3个交易日**再执行新买入 |
| 冷静期内出现新⭐⭐⭐⭐⭐信号 | 可提前介入，但需满足：①通过相关性过滤 AND ②BIAS20<5% AND ③分市场相对强弱全部通过 |
| 止损后无信号可选 | **空仓等待**，不强制填满5只 |

### 10.4 消息不能替代技术信号

> ⚠️ **核心原则：消息面辅助决策，但技术信号是最终执行依据。**

- ❌ **不能因为消息利好就买入**：再大的利好，技术面没到位也不买
- ❌ **不能因为消息利空就不卖**：技术面已破位，必须执行卖出
- ✅ **消息的作用是调整仓位大小和执行速度**，不是改变买卖方向

**例外（技术信号被消息推翻）：**
- 重大系统性风险（上述利空表格中的🔴事件）→ 技术面未触发也执行卖出
- 这类事件平均每年 1-3 次，不影响整体策略胜率
